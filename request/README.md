### 需求更新：

当需求更新完成时候 在对应要求后面划√，具体接口文档在api里面撰写

用户相关：

1. 获取所有用户名单和信息
2. 用户姓名和用户密码录入
3. 通过用户id获取单个用户所有信息
4. 通过用户id修改用户的信息（uid无法修改）
5. 通过用户id判断是user还是admin
6. 通过用户id获取用户身份是有学生
7. 通过用户id获取用户身份会员等级
8. 通过用户id获取用户下单订单
9. 通过用户id获取用户购物车
10. 通过用户id注销用户（将status = 0）


后端相关

1. 获取所有用户人数
2. 获取所有商品数
3. 获取所有订单数
4. 获取今日订单数（局限时间）
5. 获取今日新增用户数（局限时间）
6. 获取所有订单总的销售额
7. 获取销售趋势（按照天进行划分，给出一个月内每日订单个数）
8. 获取用书增长趋势（按照天进行划分，给出一个月内每日用户增长）
9. 获取最近活动（当用户注册，商品上新，用户评论时，显示最近10条记录并存储）
10. 用户禁用选项
11. 获取所有用户信息（ID，用户名，手机号，邮箱，状态，注册时间）
12. 获取所有商品信息（ID，商品名称，价格，库存，CPU，GPU，评分）
13. 获取所有订单信息
14. 获取所有商家信息
15. 获取对应商品对应评价（商品ID，评价评分，评价时间，评价内容）
